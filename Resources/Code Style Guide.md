# C++ Code Style Guide

## General
- Use meaningful variable and function names
- Keep functions short and focused
- Comment your code where needed

## Indentation & Formatting
- 4 spaces per indentation level
- Use `{}` braces for all control structures, even single-line
- Put `else` on the same line as the closing brace of `if` if possible

## Functions
- Each function should do one thing
- Use camelCase for function names: `calculateAverage()`

## Variables
- Use camelCase for variables: `studentScore`
- Constants in ALL_CAPS: `MAX_STUDENTS`

## File Naming
- Use lowercase with underscores: `week01_task.cpp`

## Git Commit Messages
- Short, descriptive messages
- Use present tense: `Add function to calculate factorial`
